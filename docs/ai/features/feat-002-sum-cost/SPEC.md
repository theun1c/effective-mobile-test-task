# SPEC.md

## Фича

Подсчёт суммарной стоимости подписок.

## Цель

Реализовать HTTP-ручку для подсчёта общей стоимости подписок за выбранный период.

## Описание

Фича считает сумму расходов на подписки за период.

Расчёт должен поддерживать фильтрацию:

- по ID пользователя;
- по названию подписки.

## API

```http
GET /subscriptions/total
```

## Query-параметры

| Параметр | Обязательный | Описание |
|---|---:|---|
| `from` | да | Начало периода в формате `MM-YYYY` |
| `to` | да | Конец периода в формате `MM-YYYY` |
| `user_id` | нет | Фильтр по пользователю |
| `service_name` | нет | Фильтр по названию подписки |

## Пример ответа

```json
{
  "total_cost": 2400
}
```

## Бизнес-правила

- Подписка участвует в расчёте, если активна хотя бы один месяц внутри выбранного периода.
- Если `end_date` не указана, подписка считается активной после `start_date`.
- Стоимость считается по месяцам.
- Количество месяцев считается включительно.
- Итоговая сумма возвращается целым числом в рублях.

## Валидация

- `from` — обязательный параметр в формате `MM-YYYY`.
- `to` — обязательный параметр в формате `MM-YYYY`.
- `to` не может быть раньше `from`.
- `user_id` должен быть валидным UUID, если передан.
- `service_name` не должен быть пустым, если передан.
