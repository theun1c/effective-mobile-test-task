# SPEC.md

## Фича

Управление подписками.

## Цель

Реализовать REST API для работы с записями о подписках пользователей.

## Описание

Фича отвечает за базовые операции с подписками:

- создание подписки;
- получение подписки по ID;
- получение списка подписок;
- обновление подписки;
- удаление подписки.

Каждая подписка содержит:

- название сервиса;
- стоимость месячной подписки в рублях;
- ID пользователя в формате UUID;
- дату начала подписки;
- опциональную дату окончания подписки.

Проверка существования пользователя не требуется.

## API

```http
POST   /subscriptions
GET    /subscriptions
GET    /subscriptions/{id}
PUT    /subscriptions/{id}
DELETE /subscriptions/{id}
```

## Модель данных

```json
{
  "id": "uuid",
  "service_name": "Yandex Plus",
  "price": 400,
  "user_id": "60601fee-2bf1-4721-ae6f-7636e79a0cba",
  "start_date": "07-2025",
  "end_date": "12-2025"
}
```

## Валидация

- `service_name` — обязательное поле, не пустая строка.
- `price` — обязательное поле, целое число больше 0.
- `user_id` — обязательное поле, валидный UUID.
- `start_date` — обязательное поле в формате `MM-YYYY`.
- `end_date` — необязательное поле в формате `MM-YYYY`.
- `end_date` не может быть раньше `start_date`.
